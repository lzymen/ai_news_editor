# 要求和思路

## 要求

用户给一个标题，AI 生成对应标题的新闻文稿

## 思路

创建多个 AI Agent 共同参与完成

### 标题Agent

按照用户给出的标题，进行需求分析，对标题进行润色，重新制定标题

然后将新的标题作为所有 Agent 都能访问的数据

### 信息收集Agent

通过新闻标题和内容大纲，拆分想要搜索的关键字，并搜索资料

### 大纲生成Agent

通过标题和搜索结果制定新闻内容大纲

### 段落撰写Agent

根据内容大纲，并参考搜索资料，撰写具体的段落内容

### 校对Agent

校对事实性，根据 信息收集Agent 搜索的资料，校对内容是否真实合理

### 评论Agent

对校对通过的内容进行评论，给出修改意见

### 总结排版Agent

将写好的文章段落拼接起来并进行排版

### 总流程

采用 **消息** 和 **循环** 驱动

1. 标题Agent 制定标题
2. 信息收集Agent 通过标题拆分出查询关键字，查询搜索资料
3. 大纲生成Agent 通过标题和搜索结果制定新闻内容大纲
4. 段落撰写Agent:
   1. 当他接到大纲生成Agent提供的大纲，则根据内容大纲，并参考搜索资料，撰写具体的段落内容
   2. 接到了校对Agent重写段落内容的请求，则根据事实性，修改不真实的部分
   3. 接到了评论Agent的修改意见，则根据意见修改段落内容
5. 校对Agent 校对真实性，若出现与事实不符的内容，则跳转到步骤 4，让段落撰写Agent 重新撰写
6. 评论Agent 对校对通过的内容进行评论，给出修改意见，并将意见重新给到第 4 步，段落撰写 Agent 考虑意见，进行修改
   1. 评论Agent通常会不停的提意见，所以永远不会结束，则我们设置一个最大循环次数
7. 评论 Agent 的意见修改结束后，我们重新从步骤 4 开始，撰写下一段内容
8. 直到 段落撰写Agent 的所有段落撰写完成，通知 文章总结Agent，总结文章并输出结果
9. 文章总结Agent，将所有段落拼接一起，并输出最终的文章内容，返回一个程序退出的信号
